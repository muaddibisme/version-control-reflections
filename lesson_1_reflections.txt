What do you think are the pros and cons of manually choosing when to create a commit, like you do in Git, vs having versions automatically saved, like Google Docs does?
-The main PRO is that you can keep each version to a logical chunk of the project. This is what is lost/obfuscated with auto-saving. The main CON is that you have to conciously make your commits.

Why do you think some version control systems, like Git, allow saving multiple files in one commit, while others, like Google Docs, treat each file separately?
-Target audience. Git is focussed on projects where multiple files are often linked whereas goggle docs is more focussed on one doc at a time.
